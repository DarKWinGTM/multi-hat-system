# Decision Mode

Use Decision Mode only for explicit comparisons.

Examples:
- "Should I use REST or GraphQL?"
- "PostgreSQL or MongoDB?"
- "Which approach is better here?"
- "Compare A and B"

---

## Core Goal

Compare a **small set of real options** through all three hats and make the decisive trade-off visible.

---

## Decision Process

1. Restate the decision and explicit options.
2. Clarify which criteria matter most.
3. Analyze each option through all three hats.
4. Identify decisive trade-offs.
5. Pressure-test disagreements.
6. Recommend one option.
7. State when another option would win under different constraints.

---

## Option-Set Rules

- prefer the user's explicit options when provided
- keep the set small and real
- do not force 3 options when only 2 real options exist
- if the option framing is incomplete, add at most one clearly justified missing alternative

---

## Cross-Hat Challenge Step

After option analysis, explicitly ask:
- where do the hats agree?
- where do they disagree?
- which disagreement matters most?
- what trade-off is being accepted in the final recommendation?
- is a hybrid genuinely better or only more complicated?

---

## Good Decision Output Should Show

- why the winner wins
- what trade-off is accepted
- what context would flip the recommendation
- what implementation or adoption considerations follow from the choice

---

## Decision Anti-Patterns

- compare options when the user did not ask for comparison
- force exactly 3 options when only 2 real options exist
- choose based on trendiness rather than fit
- skip trade-off explanation
- use fake debate language with no substantive comparison
